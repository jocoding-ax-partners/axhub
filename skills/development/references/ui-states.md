# UI 상태 + 스타일 정합 (E2)

development 가 생성하는 read 화면은 happy path 만 만들지 않아요. vibe coder 앱이 실데이터에서 깨지지 않게 3상태를 항상 넣고, 기존 앱 톤에 맞춰요.

## 3상태 (항상)

- **loading**: 데이터 가져오는 동안 스켈레톤/스피너. 빈 화면 깜빡임 금지.
- **empty**: 결과 0건일 때 "아직 데이터가 없어요" 류 안내 + 다음 행동 힌트 (빈 배열을 그냥 렌더해서 빈 표만 보이게 두지 않아요).
- **error**: 조회 실패 시 사람이 알아들을 한국어 메시지 + 재시도 경로. raw 에러·스택은 화면에 안 띄워요.

## 스타일 정합

- 기존 앱의 컴포넌트·디자인 토큰(색·간격·타이포)을 먼저 읽어 같은 스타일로 만들어요 (앱 규약 학습 단계 결과 활용). 새 디자인 시스템을 즉석에서 만들지 않아요.
- DESIGN.md 가 있으면 그 제약을 따라요.

## 대용량·경계

- 큰 결과(예: 수천 행)는 페이지네이션/가상 스크롤을 넣어요 — 전부 한 번에 렌더 금지.
- 긴 문자열·null·예상 밖 타입에 화면이 안 깨지게 방어적으로 렌더해요.

## 접근성 기본

- 키보드 포커스·대비·터치 타깃 기본은 챙겨요. 표·목록은 의미있는 헤더를 줘요.
