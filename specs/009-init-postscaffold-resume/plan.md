## Summary

`init` 은 이미 성숙한 scaffold saga 스킬이에요 (CLI확인→template→이름→dry-run→사용자 확인→`apps bootstrap` 실행→device-flow→clone→infer-tables-env 핸드오프). 현재 동작을 baseline 으로 고정하고, vibe coder 체감이 가장 큰 두 지점만 선택적으로 확장해요.

- **A. 생성 후 자동 연결**: clone 직후 의존성 설치 + dev 서버 background 실행 + 결과 안내까지 자동. 결과는 **배포 공개 URL 을 hero** 로 (vibe coder 의 진짜 마법), localhost 는 보조.
- **B. 세션 끊겨도 복구**: bootstrap 진행 상태를 저장해, 중단돼도 **재진입 시 자동으로 "이어서 할까요?" 를 먼저 제시**. magic phrase 몰라도 복구돼요.

**핵심 구현 원칙**:
1. **All-Rust** — 새 shell glue 를 Rust `axhub-helpers` 서브커맨드로 (bash/jq 는 Windows-native PowerShell 에서 깨짐, spec 004 FR-001/002). cargo test 1회 = cross-platform 커버.
2. **Vibe coder safe + dependency contract explicit** — `init` 이 의존성 설치를 직접 유발하므로 `allows-dependency-execution: true` + allowlist + AUQ 실행 확인 + `--ignore-scripts` 를 같이 추가해요. 사용자에게 CLI 명령(`npm i` 등)을 보여주지 않아요. degrade/error 는 자연어 복구("다시 해줘"/"도와줘")만, raw 명령은 `AXHUB_INIT_VERBOSE` 뒤로 (기존 패턴).
3. **No new remote mutation** — local 실행·상태저장은 AUQ 확인 + 멱등 + fail-soft. 기존 helper/skill 위임 재사용.

clone 성공(Step 7) 직후, infer-tables-env 핸드오프(Step 8) 앞에 삽입. 감지·실행은 helper 위임.

### A-1. 흐름

```
clone 성공 (Step 7)
      │
      ▼
[A-1] axhub-helpers scaffold-detect --json   (pm + node + dev-script)
      ├─ package.json 없음 → skip → Step 8 (배포 URL hero)
      ├─ lockfile 없음 → install/dev skip → Step 8 (배포 URL hero, "코드는 준비됐어요")
      ├─ node 없음 → "앱 실행 준비를 도와드릴게요" → onboarding 위임 (CLI 명령 노출 안 함)

      │ "네, 실행까지"
      ▼
[A-3] axhub-helpers scaffold-dev start --json   (자연어 narrate "앱 준비 중이에요")
      │  내부: <pm> install --ignore-scripts → dev spawn(background) → localhost URL 파싱 → PID/port/process-group state (멱등, E1/E5)
      ├─ 실패(install/dev/포트/URL) → 자연어 degrade ("미리보기는 잠시 후에 다시 해줘"); raw 명령 VERBOSE 뒤 (D1)
```

### A-2. 결과 카드 (D2 magical moment + E2 정합성)

auto-connect 실행 여부와 무관하게 **배포 공개 URL 이 hero**예요. 단, URL 은 절대 합성하지 않아요. source 우선순위:

1. terminal bootstrap/status 응답의 deployment/public URL
2. `axhub deploy status` 또는 `axhub open` 결과 URL
3. URL 이 없으면 dry-run 의 subdomain label 은 내부 힌트로만 보관하고, 사용자에게는 "인터넷 배포가 시작됐어요" 카드로 낮춤

URL 이 확인된 경우:

```
🎉 인터넷에 올라갔어요: <confirmed-public-url>
   친구한테 바로 보여줄 수 있어요.

(auto-connect 성공 시 추가)
직접 고치면서 보고 싶으면: 로컬 미리보기 http://localhost:<port> 도 떠 있어요.

다음에 뭐 할까요?
- 코드 고치고 "다시 배포해줘"
- "방금 배포 어디까지 됐어?"
```

- dry-run 의 `metadata.subdomain` 은 URL 이 아니므로 인터넷 주소처럼 임의 조합하지 않아요.
- 기존 SKILL.md:263-268 의 "1. 의존성 설치 npm i / 2. 로컬 실행 npm run dev" **수동 CLI 안내 제거** (D1+E2) — auto-connect 가 했거나, 안 됐으면 자연어 재개로.
- infer-tables-env 핸드오프는 이 카드 다음 유지.

## B. Cross-session Resume (init Step 0.5 proactive + Step 6/7 상태저장)

### B-1. Resume state (E3 — 최소, bootstrap-status 권위)

기존 `state-update`/`state-show` 는 품질 state 전용이라 쓰지 않아요. 기존 `.axhub/bootstrap.state.json` 도 `bootstrap-state/v1` version 검증이 있어 섞지 않아요. 새 helper `init-resume` 이 repo-local `.axhub/init-resume.json` 을 원자적으로 R/W 해요. **stage 저장 안 함.**

```json
{
  "schema_version": "init-resume/v1",
  "template": "<template-alias-or-id>",
  "app_name": "<human-name>",
  "slug": "<slug>",
  "subdomain": "<subdomain-label|null>",
  "idempotency_key": "<uuid-v4>",
  "bootstrap_id": "<uuid|null>",
  "repo_full_name": "<owner/repo|null>",
  "clone_done": false,
  "pending_device_flow": false,
  "created_at": "<iso8601>",
  "updated_at": "<iso8601>"
}
```

- **write 순서 (E3)**: (1) Step 6 execute 직전 breadcrumb (`template/app_name/slug/subdomain/idempotency_key`, `bootstrap_id:null`). (2) device-flow event 후 fast-exit 하면 `pending_device_flow:true`. (3) execute/status 응답에서 `bootstrap_id` + `repo_full_name` backfill. (4) clone 성공 시 `clone_done:true` 후 clear. raw 값 echo 금지 (Visibility Rules).
- **clear**: clone 성공/명시 abort 시 삭제 (멱등). 보존: clone `git reset --hard` 는 tracked 만.
- **gitignore**: `.axhub/init-resume.json`, `.axhub/scaffold-dev.json`, `.axhub/scaffold-dev.log` 를 idempotent append. 기존 `.axhub/bootstrap.state.json` entry 와 별개.

### B-2. 재개 진입 (D3 proactive — Step 0.5)

```
init 진입 (기존 trigger "앱 만들어줘" 등) → init-resume route --json: (bootstrap_id 존재 OR pending_device_flow) & clone 미완 ?
   │ 예 (재진입 자동 감지 — magic phrase 불필요)
   ▼
"저번에 만들던 앱(<slug>) 이 있어요. 이어서 할까요?" (AUQ; D1 비대화형 → safe_default skip)
   │ 네
   ▼  helper route enum 이 현재 saga 상태를 조회 (stored stage 안 믿음, E3):
   ├ saga in-flight → bootstrap-status --watch --watch-timeout 9m
   ├ pending_device_flow/bootstrap_id 없음 → bootstrap --template "$TEMPLATE" --name "$APP_NAME" --slug "$APP_SLUG" --execute --resume-last --watch --watch-timeout 9m --idempotency-key "$IDEMPOTENCY_KEY"
   ├ device code 만료/캐시 없음 → 같은 state 값으로 fresh execute 재시도(사용자가 "이어서" 선택한 경우만)
   ├ terminal-success & clone_done=false → Step 7 clone 재실행
   └ status 404 / corrupt / 만료 → state 정리 + 자연어로 "새로 시작할까요?"
```

라우팅·검증은 helper(Rust): state 읽기 + staleness + pending device-flow 여부로 route enum 을 만들고, `bootstrap_id` 가 있으면 `status_command` 를 함께 반환해 bootstrap-status 를 권위 source 로 쓰게 해요. SKILL 은 enum/command 를 받아 실행하고 자연어로 narrate 해요. `bootstrap-status` 를 부를 수 없는 상태(`bootstrap_id:null`) 에서는 helper 가 `resume_command` 를 반환하므로 `--resume-last` 를 먼저 시도하고, 실패 reason 이 `no_cached_device_flow`/`expired` 일 때만 fresh execute 로 넘어가요.

### B-3. RESOLVED (D3 로 해소)

이전 Eng-review 의 open landmine ("이어서 해줘" → init 도달 시키려면 `description:` trigger 추가 → keyword baseline 충돌) 은 **D3 로 해소**됐어요. 재진입 시 **기존 trigger** 로 들어와 Step 0.5 가 proactive 하게 복구를 제안하므로, **새 trigger 도 baseline 재캡처도 불필요**. (full session-start hook surfacing 은 P3 TODO 로 잔존 — 어떤 발화에도 복구 제안하려면.)
