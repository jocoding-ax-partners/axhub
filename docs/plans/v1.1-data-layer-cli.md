## 1. 핵심 재해석 — "MCP tool" = CLI 호출

spec/guide가 MCP tool이라 부르는 것을 이 plan에선 전부 CLI로 읽어요. claude-code(에이전트)가 Bash로 `axhub catalog ...`를 직접 실행하고, `.axhub/AXHUB.md` + skill description이 그 행동을 규제해요 (= 기존 "tool description으로 AI 규제"의 대체).

| 문서의 MCP tool | 실제 구현 = CLI / 플러그인 |
|---|---|
| `axhub.search` | 에이전트가 `axhub catalog search --json --limit 200 [--kind --connector]` 실행 |
| `axhub.describe` | `axhub catalog get --connector <c> --path <p> --json` |
| `axhub.invoke` | `axhub catalog invoke --connector <c> --path <p> --action read --sql <s> --row-limit <n> --execute --json` |
| `axhub.snippet` | `axhub-helpers snippet ...` (신규 helper 서브명령, 코드 텍스트 생성 — backend 무관) |
| `plugin sync` | `axhub-helpers sync ...` (신규 helper 서브명령, `.axhub/` 생성) |
| "tool description으로 AI 규제" | `.axhub/AXHUB.md` 본문 + `skills/data/SKILL.md` description |

### ⚠️ 핵심 정정 — `catalog invoke`는 `--execute` 필수 (agent-safe 기본값)

검증 결과: `axhub catalog invoke`는 **read여도 default가 dry-run**이에요.

```
$ axhub catalog invoke --connector x --path y --sql "SELECT 1" --json
{ "schema_version":"1", "status":"dry_run",
  "data": { "dry_run_caveats": ["client-side simulation only — backend validation not performed", ...] } }
```

→ 실제 데이터를 받으려면 `--execute`를 붙여야 backend에 가요 (`refactor/agent-safe-cli-skills` 철학 = mutate-by-default 금지). guide §3.3/§5.6의 "plain invoke가 결과 반환" 예시는 **CLI 기준 틀려요**. 그래서:

- `.axhub/AXHUB.md`의 invoke 예시 + `target=local-bash` snippet + 에이전트의 실제 read 호출은 **항상 `--execute --json`** 을 써요.
- Python/TS/Go snippet은 CLI가 아니라 backend `:read` 엔드포인트로 HTTP POST하므로 영향 없어요 (아래 §9 구분 참고).

## 3. 데이터 흐름 + shadow paths

### 3.1 `sync` 흐름 (happy + 4 shadow)

```
INPUT(cwd, --target?) ─▶ axhub catalog resources --json --limit 200
                              │
        ┌──────── happy ──────┼──── nil ──── empty ──── error ────┐
        ▼                     ▼              ▼                     ▼
  per-hit:               (--target 없고     (resources []     (CLI exit≠0:
  axhub catalog get      heuristic 모호)     = 권한 0)          auth/네트워크)
  --json                 → AXHUB_TARGET      → catalog.json     → classify()로
        │                   prompt (skill)    resources:[] 로     한국어 에러,
        ▼                   또는 기본값       정상 생성 +         .axhub 미생성,
  catalog.json 조립         web/local 추정    "권한 resource      exit≠0
  (mask lower-case          후 파일 기록      없음" 안내
   정규화 #143)
        ▼
  write .axhub/{AXHUB.md, AXHUB_TARGET, catalog.json}
  + ./.gitignore 에 ".axhub/catalog.json" append (PII)
```

- **nil**: `--target` 미지정 + heuristic 모호 → skill이 AskUserQuestion으로 묻고 `AXHUB_TARGET`에 저장 (guide §1.2). helper sync는 `--target auto`면 heuristic, 모호하면 `"ambiguous"` status 반환 → skill이 prompt.
- **empty**: `resources: []` (권한 0) → catalog.json 정상 생성, 빈 배열. AI에게 "권한 resource 없음" 안내. panic 금지.
- **error**: CLI 비-0 exit → `classify-exit`로 한국어 매핑, `.axhub/` 미생성 (부분 쓰기 금지), exit 전파.
- **[Eng review issue 3 fold]** 쓰기 위치 = git toplevel (`git rev-parse --show-toplevel`) 기본, `--out <dir>` override. CWD가 서브디렉터리여도 프로젝트 루트에 `.axhub/` 생성.
- **[Eng review D2→A + D5/codex #9] staleness 정책**: `skills/data`가 진입 시 `catalog.json` `generated_at` 확인. **`.axhub/` 없으면(최초) → 생성+`.gitignore` 편집 전 consent 묻기** (mutating bootstrap). **이미 있으면(opt-in 됨) → stale(threshold session 1회/1h) 시 silent re-sync** (data만 갱신). agent-safe 유지 + stale 500 예방.
- **[Eng review perf note]** per-resource `get`은 tokio bounded-concurrency join으로 병렬화 (N+1 완화, 선택적). `--no-detail`로 detail 생략 fast sync도 지원 (spec §6).

## 4. Error & Rescue map (#155–164 → CLI → 한국어)

`axhub catalog` CLI는 `--json` 에러 envelope `{code, category, message, retryable, request_id}` + exit code를 줘요.

**[Eng review D1 → B 정정]** `classify()`의 에러 사전(`error-empathy-catalog.md`)은 현재 **deploy/github/apps 코드만** 담고 catalog-API 코드(`internal_error`/`not_found`/deny/sql_format)가 없어서 그대로론 `default_entry`("알 수 없는 에러")만 나와요. 따라서:
1. **[codex #2]** `crates/axhub-helpers/data/catalog.json` (source-of-truth)에 `catalog.*` 엔트리 추가 → `bun run codegen:catalog` regen (markdown은 generated 산물 — 직접 편집 X).
2. **비-0 exit 에러**(404/500/401): `axhub catalog ... --json; echo $?` → 기존 `classify-exit` 통과 → 결정적 한국어. classify 키는 `<exit_code>:<error.code>`라 그 조합으로 등록.
3. **[codex #3] deny(HTTP 200, `allowed:false`)는 exit 0** → classify-exit가 `{}` 반환(매핑 안 됨). skill/helper가 응답 body의 `allowed`/`deny_reason`을 **직접 파싱**해서 §4 표대로 안내. (classify-exit는 deny adapter가 아님.)
4. golden test로 비-0 코드(internal_error/not_found/sql_format/401) + deny body-parse 분기 고정.

v1.1 backend 변경을 아래대로 분기해요.

| backend (PR) | CLI 신호 | 에이전트/사용자에게 | rescue |
|---|---|---|---|
| `allowed:false` generic deny (#162) | HTTP 200 body `allowed:false`, `deny_reason` generic | deny_reason 그대로. "다른 SQL/우회 시도 금지" | 표시만. 재시도 X |
| SQL 형식 오류 | `allowed:false`, `deny_reason:"SQL 형식..."` | deny_reason 그대로 | 사용자 SQL 수정 유도 |
| catalog 미등록 column → 500 (#156) | HTTP 500 `internal_error` | "SQL column을 `axhub catalog get`의 allowed_columns와 비교. **retry 금지**" | **자동 retry 절대 금지** (영구 에러 가능) |
| denied detail path → 404 (#163) | HTTP 404 `not_found` | "리소스 못 찾음. `axhub catalog search`로 재확인" | search로 재발견. path 추측 금지 |
| admin governance 차단 (#164) | `/grants`,`/tags`,`/subjects` = 403/미노출 | skill/AXHUB.md가 애초에 호출 안 하게 막음 | 노출 안 함 (예방) |
| column deny invisible (#155) | invoke 응답에 권한없는 column 부재 | snippet은 catalog의 allowed_columns만 reference | 예방 (생성 단계) |
| mask enum (#159) + case (#143) | catalog detail의 mask 값 | sync가 lower-case 정규화 후 catalog.json 저장 | 정규화 |
| 401 | `not authenticated` exit 4 | Mode A: SSO 재로그인 / Mode B: `/axhub:auth` 안내 | 재인증 라우팅 |

**catch-all 금지**: classify는 exit_code + error code별로 분기. `_ => "알 수 없는 에러"`는 default_entry로만 (이미 존재). 모든 알려진 #155-164 케이스는 명시 분기.

## 5. Hook & consent 통합 (이 브랜치의 핵심 레이어)

이 PR은 `refactor/agent-safe-cli-skills` 위에 올라가요. 새 surface가 기존 hook 계약을 지켜야 해요.

- **consent 게이트**: `cmd_preauth_check` + `parse_axhub_command`(`crates/axhub-helpers/src/consent/`)가 **mutation** 서브명령(apps create / github connect / auth login / deploy)에 consent-mint를 요구해요. `catalog invoke --execute`는 `action=read` = mutation 아님 → consent-gate 대상 아님으로 설계.
  - **회귀 검증**: `consent/`의 destructive 집합에 `catalog`가 없는지 확인하고, `catalog invoke --execute` read 호출이 mutation consent 로 분류되지 않는 regression test 를 유지해요. read는 게이트하지 않되, `--execute` 명시성 자체가 agent-safe intent 신호 (dry-run 기본과 일관).
- **hook-safety**: 새 subcommand는 hook이 아니라서 fail-open 계약 직접 대상은 아니지만, skill의 in-body preflight(`needs-preflight`)가 CLI 존재·버전·auth를 확인해요.
- **AXHUB.md가 에이전트에게 고지할 것**: (1) invoke는 `--execute` 필요, (2) governance 엔드포인트(`/grants` 등) 호출 금지, (3) denied/404/500 응답 처리 규칙 (§4).

## 9. `.axhub/` 계약 + snippet 계약

### 9.1 `.axhub/` (git toplevel 기본 생성, `--out <dir>` override)

```
.axhub/
  AXHUB.md       ← assets/AXHUB.md 그대로 (정적, spec §5 본문)
  AXHUB_TARGET   ← "web-axhub" | "local-python" | ... (heuristic+prompt)
  catalog.json   ← 권한 resource snapshot (§6 형식, allowed_columns 포함)
```
- **[Eng review D3 → A]** `AXHUB.md` canonical = `crates/axhub-helpers/assets/AXHUB.md` (commit), `include_str!`로 임베드. `docs/plugin-developer-guide.md §4` 본문은 "본문은 `crates/axhub-helpers/assets/AXHUB.md` 참조" **포인터로 교체** (단일 소스, drift 불가). 두 벌 유지 안 함 → 동등성 테스트 불필요.
- sync가 사용자 프로젝트의 `./.gitignore`에 `.axhub/catalog.json` append (user_email/tenant_id PII). `AXHUB.md`/`AXHUB_TARGET`은 commit 가능.

### 9.2 snippet (runtime 사용자 코드 = HTTP, CLI 아님)

- 매트릭스: Mode A → `typescript`(web-axhub); Mode B → `python`/`typescript`/`go`/`shell`.
- 템플릿 = `crates/axhub-helpers/templates/{mode-a-ts,mode-b-python,mode-b-ts,mode-b-go,mode-b-shell}.tmpl`, `include_str!`.
- Mode A: `credentials:'include'` + relative path, **Authorization 헤더 없음**.
- Mode B: **`X-Api-Key: <PAT>`** ([codex #4] CLI auth skill 기준 PAT=X-Api-Key. guide §1.3/spec §5의 `Authorization: Bearer` 서술은 구현 중 정정), env var만 **하드코드 없음**.
- `target=local-bash`만 예외적으로 `axhub catalog invoke --execute` CLI 형태도 선택 가능 (CLI 설치 가정 시). 기본 shell = curl + env PAT (guide §1.3).
- 상단 주석 표준 (guide §1.5): `mode=A|B · target · allowed_columns · masked`.
- **[Eng review issue 5 fold]** placeholder 치환 = 명시적 minimal substitution (`{{TENANT}}`/`{{CONNECTOR}}`/`{{PATH}}`/`{{SQL}}`/`{{ALLOWED_COLUMNS}}`/`{{MASKED}}`). template crate(tera 등) 안 씀 (over-engineering 회피). 임베딩되는 SQL·path는 **언어별 escaper** 통과 (Python triple-quote/repr, TS template-literal `${}`·backtick escape, Go raw-string `\`` 회피, shell single-quote). 언어별 escaping unit test 필수.
