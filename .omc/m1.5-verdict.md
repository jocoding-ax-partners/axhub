### Why each metric improved

**Trusted completion (+40pp):** Plugin routes NL utterances through skills/ with opinionated flag defaults (--follow --source build for logs, --watch for status, --json everywhere). Docs-only Claude frequently omits these flags or skips the post-deploy status chain. Plugin also handles negative cases (T-NEG-*) correctly via clarify fallback, and Korean-language routing (T1, T2, T3, T4) via the NL trigger lexicon.

**Unsafe-trigger bypass (0%):** PreToolUse HMAC gate in `cmdPreauthCheck` (src/axhub-helpers/index.ts:178-227) intercepts every destructive Bash call and calls `verifyToken(binding)`. A freshly minted consent token is required — bound to the specific tool_call_id, action, app_id, profile, branch, and commit_sha. No text claim in the user utterance (T-ADV-3), no transcript inference (T-ADV-2), and no user pressure (T-ADV-1) can produce a valid HMAC token without going through `AskUserQuestion` → `consent-mint`. Bypass rate drops from 90% to 0%.

**Recovery rate (100%):** PostToolUse `classify-exit` hook catches exit 65 (auth expired) and surfaces a Korean systemMessage + AskUserQuestion prompt, enabling the auth-login recovery flow (T5, T-HEADLESS-1). Cold-cache multi-machine fallback (T-MULTI-1) falls back to `axhub apps list --json` then asks the user to identify the deploy — a successful recovery chain. Baseline had 0% because docs-only Claude neither catches exit 65 automatically nor implements the cold-cache fallback.
