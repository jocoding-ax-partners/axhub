### Learnings

- Sub-agent 1M context inheritance fails when parent has the flag set; spawned executors error with "Extra usage required for 1M context". Workaround: do work directly OR launch agents in fresh sessions. Phase 2 had 2 agents fail this way (US-104 + US-105 sonnet) → recovered by inline implementation.
- Generator + serialized .json contract pattern works well for "frozen by design" testing. README + commit-both protocol prevents drift. Bonus: external tools can consume .json directly without TS toolchain.
- Bun cross-arch compile is mature: `bun build --compile --target=bun-{os}-{arch}` produces working binaries for 5 targets in <2s each. Windows binary largest at 109.6M (PE32+ runtime overhead) — 120MB threshold appropriate.
- Phase 6 plugin-validator agent pre-empted bug class A (manifest shape) but missed real loader edge cases (object repository, missing hookEventName). Real loader testing is non-substitutable. Persistent assertions in tests/manifest.test.ts are the regression gate.
- Mulberry32 PRNG (10 lines, 32-bit period) is sufficient for fuzz reproducibility. Don't reach for crypto-grade RNG when seed-determinism is the only requirement.
- Telemetry opt-in default-OFF posture is the right tradeoff for B2B vibe-coder tool — surprise data collection in any session would erode trust faster than the analytics value.
