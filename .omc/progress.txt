### Learnings

- Recursive collectCommandPositions with depth-limit 5 + shell-string body unescape (`\"` → `"`) closes the eval-of-bash gotcha class. Pattern: when extracting a quoted body, the outer shell already processed escapes; a parser must mirror that to see what would actually run. Without unescape, `eval "bash -c \"axhub deploy create\""` looks like 2 separate strings to the parser instead of 1 destructive command.
- Hand-written + auto-generated catalog parallel files (instead of replacing hand-written) preserves placeholder documentation while still adding regression-detection coverage. Drift test (corpus-schema.test.ts pattern: validate symmetry without forcing single-source) is the better architecture than "regenerate everything".
- Bun cross-compile (build:all) replaces Go's goreleaser cleanly — no need for a separate config file when the build matrix is 5 explicit script entries. Goreleaser's value is binary uploads + GitHub Releases automation, both of which softprops/action-gh-release handles natively. Document the substitution to prevent future "where's .goreleaser.yml?" confusion.
- Phase 6 plugin-validator pre-empts JSON shape bugs but cannot pre-empt actual loader edge cases. The 295-test gate (with manifest.test.ts asserting hookEventName presence + repository-as-string + cross-consistency) is the only sustainable defense — agent recommendations need real-loader cross-validation.
- Out-of-scope-but-shipped pattern: US-205 + US-206 ship infrastructure (docs + scaffolds) for human-coordination tasks (customer recruit + cred procurement). Ralph CAN'T do these autonomously, but it CAN reduce the "last mile" friction by shipping ready-to-use templates. Honest documentation of out-of-scope items in each file's header prevents over-promising.
- describe.skipIf pattern (with positive + negative paired blocks) gives clearest test runner output for env-gated suites. The 5-line placeholder describe is worth its weight — turns "empty file run, 0 tests" into "1 placeholder pass, gating reason printed to stderr".
